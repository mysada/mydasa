Request for Proposals

Capabilities Demonstration

1. Introduction

Our company (WDD Development) has been asked to demonstrate its capabilities in developing eCommerce solutions. A local marketing firm, XYZ Marketing, has acquired a number of recent clients who are looking to develop eCommerce websites or web Applications. XYZ is primarily a communications and design studio, and has no wish to build an in-house programming/development department. As a result, XYZ is looking to partner with a local firm in handling their clients needs.

Our firm has been shortlisted, along with two others. We have already shared our capabilities document with XYZ, but the management team has agreed that we would like to put together at two to three eCommerce solutions to act as live examples of our capabilities. As a result, we have created internal teams comprising our top developers (you) to handle this.

Each team is being asked to develop a modern, fully functioning eCommerce solution. Each solution should demonstrate our capabilities as a full-stack development studio. Although it is most likely that XYZ would provide some of these services when outsourcing to us, we feel it is important to demonstrate our own in-house capabilities in each area:

  • Project Planning
  • Graphic Design
  • Data Modelling and database management
  • UI/UX design
  • Server deployment and setup
  • Front end coding and programming
  • Back end programming and coding
  • Team/Distributed work capability
  • Git usage
  • Professionalism in all aspects of our work

2. The Teams

Each team comprises individuals who, combined, have all the skills required to create a great project. Please keep in mind, that there may be a wide range of skills and interests among team members; it is the purpose of this project to show that all team members can contribute to their full capabilities to any project they are assigned to. Teams will be announced in the kickoff meeting.

2. The project

We are asking each team to come up with a project idea. Your project may revolve around a single product type, or multiple product types. Your project may be a social selling site (think ebay), or it may be for an imaginary online retailer. Whatever type of project you come up with, you will need to meet the following requirements.

  • Fully planned and explained in a proposal/memo
  • Original graphic design (prior to coding HTML)
  • Modern, responsive, valid HTML5 and CSS3
  • Fully normalized data model (3NF)
  • Database implemented in MySQL
  • Javascript integration
  • Back end programming in PHP
  • Use of a backend MVC framework
  • Version control using Git and Git workflow for branching, pull requests, merges, etc
  • Deployment on a fully provisioned remote server (client will provide server)

Note: Before beginning ANY work or planning, please explain your idea to your supervisor (me) to get it approved.

3. Proposal

You must write a full proposal (internal document, not for outside consumption), so that management can approve and/or provide feedback prior to the beginning of work.

We have put together an outline of the proposal you should put together. Please don’t feel the need to invest too much time creating a designed, formatted document as we would for a client. This is an internal document, for internal use only.  It should be created using markdown, allowing all team members to contribute their section in text, using Git.

  1. Introduction - briefly describe the project you would like to create and how it meets our needs
  2. Project details - outline the functionality you intend to develop (both front and back end), in detail
  3. Propose design solution - Exports of your PSDs/Figma/Sketch files showing your proposed design… at least three pages please: home, list, detail and perhaps a generic page
  4. User Personas/Use Case diagrams or statements - Describe each type of user your application can accommodate… eg: guest, authenticated, editor, contributor, admin, etc. Describe how each type of user can interact with the site… walk through a typical session of each one.
  5. Sitemaps - Detailed sitemaps for front end and back end (admin) sides, showing all possible pages
  6. Server - Describe your server setup in detail (version, flavour), software installed, etc
  7. Security - Describe steps you have taken or intend to take to implement security at the server level, but also at the application level… for example, how do you intend to handle the usual Web application vulnerabilities like XSS, CSRF and MySQL injection?
  8. Database - Create a detailed data model (ERD) in appropriate software showing all tables, all relationships, all fields, and all data types.
  9. Value Adds - think of some value adds that you or your team is uniquely capable of adding to this project… how can we over-deliver so that we can impress XYZ with our capabilities and planning?
  10. Revised ERD for your Value Adds… if your value adds result in changes to your data model, please explain and provide a revised ERD
  11. Your team - Come up with a catchy team name, and list your team members, and the areas in which they will most likely make a significant contribution.

4. Deliverables

Each team will be required to submit the following deliverables on the dates specified. We understand that these timelines are VERY compressed, but this is required so that each team is able to begin work as soon as possible. The first three days are critical to the success of your project, and every team member must contribute to the work of delivering these items in as timely a manner as possible

  • Team Roles, Project idea pitch Day 1
  • Configured Server Day 1
  • Graphic Design Day 2
  • Proposal, HTML/CSS Templates (Front End) Day 3
  • Completed Site Day 12
  • Presentation Day 12

Additional requirements

  • Project must focus on physical items requiring shipment and delivery (no digital products)
  • Database must track quantity of each product on hand and adjust after purchase
    • quantity of item in stock should be displayed to user
    • items with no quantity can be displayed, but cart-add should be disabled
  • Project must adopt some kind of shipping cost strategy applicable to all orders
    • Could be per item
    • Could be per invoice total tier
    • Could be an original algorithm
  • Project must be world accessible, and apply taxes for each Canadian province or territory
  • A table will be required to track Payment Gateway transaction details
  • Data (all tables except pivots) must be fully manageable through CRUD
  • All code must be original and created by team members (with the exception of base Composer packages for frameworks)
  • All team members must contribute, with work shared equally

Note: additional requirements may be added through the course of the project
